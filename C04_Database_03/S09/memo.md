# S09. 논리적 모델링 - 실습

## 실전 논리적 모델링 - 시작

### 개념적에서 논리적으로의 변화
- 엔티티 -> 테이블
- 속성 -> 컬럼
- 식별자 -> 기본 키(대리 키)
- 관계 -> 외래 키 + 제약 조건
    - 일대다: 다쪽에 외래 키
    - 일대일: 보조 테이블에 외래 키
    - 다대다: 연결 테이블(대리 키 + 왜래 키 조합(복합 키))

### 참고: 논리적 모델링과 물리적 모델링, 무엇이 다를까?
- 관계형 데이터베이스별 물리적 타입 비교
    - 논리적 타입
        - MySQL: BIGINT AUTO_INCREMENT
        - Oracle: NUMBER(+ SEQUENCE)
        - PostgreSQL: BIGSERIAL

- 논리적 모델링: 데이터의 청사진 그리기
    - 관계형 데이터베이스 모델에 맞춤
    - 특정 기술에 종속되지 않게 구현

- 물리적 모델링: 실제 창고에 물건 쌓기
    - 실제 데이터베이스에 어떻게 데이터를 저장할지 구체적으로 정하는 단계
    - 특정 데이터베이스에 최적화된 형태로 구현
    - VARCHAR(50), TEXT 성능 및 효율을 따져 결정함

### 경계가 모호함
- 보통 이미 데이터베이스가 결정되는 경우가 많음
- 따라서 한 번에 같이 만듦
- 실습도 동일하게 진행
